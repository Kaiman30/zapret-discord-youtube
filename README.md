# 🚀 ZAPRET — Fixed Minecraft EU + Discord

Форк [zapret](https://github.com/Flowseal/zapret-discord-youtube) с исправленной конфигурацией для:

- **Minecraft Java Edition** (европейские сервера) — обход DPI провайдера
- **Discord** — стабильная голосовая связь без задержек 5000мс
- **Российские сервера с DDoS-защитой** — без ложных срабатываний

---

## 📋 Проблема оригинальных конфигов

### 1. Европейские сервера Minecraft — `Connection reset` на порту 25565

Can't ping ...: Internal Exception: java.net.SocketException: Connection reset

**Причина**: провайдеры блокируют Minecraft-трафик через DPI (Deep Packet Inspection), определяя характерные сигнатуры протокола. Стандартные методы обхода (fake/multisplit с TLS-паттернами) НЕ работают, потому что:

- Minecraft использует **собственный протокол** (не HTTPS/TLS)
- Первый пакет рукопожатия содержит hostname сервера в открытом виде
- C версии 1.19+ добавляется TLS-подобное шифрование через mbed TLS

**Решение**: `multisplit` + `any-protocol=1` + специальный паттерн `tls_clienthello_4pda_to.bin`, который точно совпадает с тем, что ищет DPI провайдера.

### 2. Discord — задержка 5000мс в голосовых каналах

**Причина**: когда Minecraft-правила находятся в том же процессе `winws`, они **перехватывают UDP-пакеты Discord**, что вызывает задержки. Даже при правильном порядке правил возможны конфликты на уровне драйвера фильтрации.

**Решение**: полная изоляция Discord и Minecraft в **разные процессы winws**.

### 3. Российские сервера с DDoS-защитой — блокировка подключения

**Причина**: Эти системы видят модифицированные SEQ-номера от `--dpi-desync-fooling=badseq` и `multisplit` как **SYN-flood атаку** и блокируют IP.

**Решение**: для российских серверов `--dpi-desync=none` — полностью отключаем вмешательство. DPI на локальных серверах отсутствует, защита сама справляется.

## 🚀 Как использовать

### Запуск

1. Запусти **cвою рабочую стратегию Zapret**
   - Обрабатывает: Discord, YouTube, браузеры, мессенджеры
   - **Не трогает** Minecraft-трафик вообще

2. Когда хочешь играть — **Не выключая свой обход**, запусти **`minecraft.bat`**
   - Обрабатывает ТОЛЬКО порт 25565
   - Никак не влияет на Discord

3. В системном трее будет **два окна winws**:
   - `zapret: Discord`
   - `zapret: Minecraft`

### Добавление серверов

**Европейские сервера** (нужен обход DPI):
Отредактируй `lists\list-minecraft-eu.txt`

**Российские сервера** (обход НЕ нужен):
Отредактируй `lists\list-minecraft-ru.txt`

**Остальные сервера** (без списка):
Автоматически применяется мягкий обход — `fake` + `badseq` с `repeats=2`

## 📊 Таблица методов обхода

| Тип серверов | Метод | Параметры | Логика |
|-------------|-------|-----------|--------|
| **ЕС заблокированные** | `multisplit` | `any-protocol=1`<br>`cutoff=n5`<br>`repeats=4`<br>`4pda_to.bin` | Агрессивный split пакетов с правильной сигнатурой |
| **РФ с защитой** | `none` | — | Полное отключение |
| **Остальные** | `fake` | `badseq`<br>`cutoff=n4`<br>`repeats=2` | Мягкий fallback без split |

## 🔧 Технические детали

### Почему `tls_clienthello_4pda_to.bin` работает с Minecraft?

DPI провайдера ищет в TCP-пакетах на порту 25565 сигнатуру, характерную для Minecraft-протокола. Файл `4pda_to.bin` содержит TLS ClientHello от сообщества 4PDA, который:

- Совпадает по длине с Minecraft handshake (~582 байта в `split-seqovl`)
- Имеет "правильную" структуру для обмана DPI
- Не вызывает reject на стороне Minecraft-сервера

### Почему два процесса winws?

Один процесс winws с правилами для портов 2053-8443 (Discord) и 25565 (Minecraft) вызывает конфликт на уровне **Windows Filtering Platform**. WFP может неправильно маршрутизировать пакеты между разными `--new` блоками, из-за чего UDP-пакеты Discord попадают под TCP-правила Minecraft.

Разделение на два процесса полностью изолирует трафик.
